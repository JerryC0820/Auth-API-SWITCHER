# 0.1.4

- 更新检查改为优先兼容旧版本强制更新场景：旧版本启动后会更快触发更新校验，不再默认等待十几秒才弹出更新拦截。
- 内测用户现在会同时检查 `beta + stable` 两条更新线，优先采用可用且版本更高的结果；当版本号相同则明确显示“当前已是最新版本”。
- 取消“同版本新修复”提示策略；版本号不变时不再反复提示更新，避免 `0.1.3 -> 0.1.3` 这种误判。
- 更新弹窗改为内嵌显示更新说明正文，不再把“查看说明”作为默认外跳路径；仍保留“打开原文”入口。
- 更新下载过程补充实时进度显示，包括顶部下载进度条、弹窗进度条、当前阶段与已下载体积反馈。
- 更新失败时新增“联系客服”入口，可直接打开客服二维码 / 联系方式 / 群入口卡片，避免用户卡在强更流程里无处反馈。
- 更新来源卡片支持悬停提示与点击展开，用户可以直接从弹窗里切到 GitHub / Gitee 仓库页面自行下载。
- 发布配置补上 Gitee release 下载基址，后续 manifest 会生成 GitHub 之外的备用下载地址。
