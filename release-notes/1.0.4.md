# 1.0.4

- OAuth 快捷登录扩展：主界面新增统一 OAuth 快捷入口，除了 Codex，还支持 Anthropic、Antigravity、Gemini CLI、Kimi 和 Qwen 的登录链路，授权完成后会自动把对应鉴权文件写入本地。
- 多 provider 鉴权落地：不同 OAuth provider 的登录结果会分别写入本地 provider 认证目录，不再只依赖单一 `auth.json`，方便后续切换和管理多套登录态。
- 高级内测权限拆分：更新通道在原有正式版和内测版基础上，补齐了“高级内测”访问能力，客户端和授权主机都会明确区分普通内测与高级内测用户状态。
- 授权主机远程通道增强：本地主机签发和复用授权码时，会保留更新通道、测试标识和高级内测权限，远程下发更新通道时也会同步写入这些状态。
- 更新源与说明地址统一：更新 manifest、下载地址和发行说明默认切到 `ainivox.cn` 域名体系，减少继续依赖旧的 GitHub 原始说明地址。
- 在线授权后端扩展：云端授权 API 增加 Ainivox 管理概览、CORS 白名单控制和新的闲鱼 CDK/订单/机器人管理接口，为后续后台运营链路打基础。
